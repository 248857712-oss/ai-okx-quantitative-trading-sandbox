4.config中的一些参数      
 leverage：杠杆倍数
 position_ratio: 每次开仓比例
 lr_weight: 逻辑回归权重
 rf_weight: 森林模型权重
 vote_threshold: 权重开仓阈值
 tp_prob_threshold: 设置这个值（止盈概率阈值），判断是否超过该止盈概率
 sl_prob_threshold: 止损概率阈值，同上
 cycle_interval: 每次执行周期（单位：s）
 boll_window: 布林窗口值
 boll_dev: 布林标准差
 min_profit_threshold: 最小止盈阈值
 target_profit_ratio: 目标收益率
 min_loss_threshold: 最小止损阈值
5.模型可以通过获得训练集重新自动调参，但当无法调参或调参回测后仍然不完美，请手动调参并回测
## **开仓逻辑（无持仓时触发）**
- 前置准备
  - 加载配置：杠杆倍数、仓位比例、概率阈值
  - 初始化客户端：连接OKX现货杠杆模拟盘
  - 训练模型：LR+RF预测上涨概率、GBSLTPModel预测止盈止损概率
- 信号生成
  - 获取1h K线数据，提取特征（涨跌幅、波动幅度、成交量比率）
  - LR/RF模型输出上涨概率，计算加权概率
  - 加权概率 ≥ vote_threshold → 生成**买入信号**
- 开仓校验（必须全部满足）
  - 布林带过滤：当前价 > 布林下轨 → 允许交易
  - 止盈止损预校验：TP概率 ≥ tp_prob_threshold 且 SL概率 ≥ sl_prob_threshold
  - 收益风险比校验：收益风险比 ≥ 1.5 且 潜在盈利 ≥ 目标盈利比例
  - 账户校验：余额充足，计算可买数量（余额×仓位比例×杠杆 ÷ 实时价）
- 执行买入
  - 发送市价买入订单（全仓杠杆模式）
  - 记录开仓价、持仓数量，更新持仓状态为1
  - 保存交易记录到本地

## **平仓逻辑（有持仓时触发）**
- 持仓监控
  - 每轮循环查询实时价，计算持仓盈亏比例
  - GBSLTPModel实时输出当前TP概率、SL概率
- 自动平仓触发条件（满足任一即执行）
  - 止盈触发：盈亏比例≥目标盈利 且 TP概率≥tp_prob_threshold
  - 止损触发：盈亏比例≤-目标亏损 且 SL概率≥sl_prob_threshold
- 手动平仓触发
  - 用户终止程序时，强制卖出全部持仓
  - 循环异常时，重试3次强制平仓
- 执行平仓
  - 发送市价卖出订单（卖出全部持仓数量）
  - 计算实际盈亏金额，更新持仓状态为0
  - 保存平仓记录到本地

## **循环逻辑**
- 每60秒执行一次轮询
- 定期重训：每24轮循环重新训练模型，适配最新行情
- 异常处理：网络/API报错时，延迟30秒重试
